# Wiki Structure Guide

This document outlines the recommended structure for the GitHub Wiki.

## Suggested Pages

1. **Home** — Project overview and quick links
2. **Getting Started** — Installation and first steps
3. **Architecture** — System design and data flow
4. **API Reference** — All endpoints (see [API_REFERENCE.md](./API_REFERENCE.md))
5. **Security Model** — Zero-knowledge guarantees, threat model
6. **Deployment** — Docker, Vercel, production setup
7. **Contributing** — How to contribute (see [CONTRIBUTING.md](../CONTRIBUTING.md))
8. **FAQ** — Frequently asked questions

## How to Enable the Wiki

1. Go to https://github.com/di3go04/zero-knowledge-vault/wiki
2. Click "Create the first page"
3. Use the content from this guide and the docs/ folder
